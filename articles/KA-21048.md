# 配置分析配置时出现内部服务器错误

## 描述 {#description}


<b>描述：</b>
尝试配置分析配置时出现内部服务器错误

<b>重现步骤：</b>

1. 转到工具 — 资产 — 分析配置
2. 输入Analytics的“分析配置”详细信息
3. 单击“Authenticate（验证）”
4. 出现错误消息“Internal server error”（内部服务器错误）弹出窗口。



## 分辨率 {#resolution}


<b>解决问题的步骤： </b>

1. 在本地实例中配置资产“分析配置”。
2. 创建路径“/etc/cloudservices/sitecatalyst/assetinsights/”的包
3. 在更高的实例中安装包。


注意：这仅适用于AMS或内部实例。
