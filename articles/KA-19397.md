# 文件夹名称为“con”的升级后问题

## 描述

<u>问题描述：</u>

<br>用户在升级实例时遇到以下错误。 无法创建文件夹 `..\var\con` 因为它已经是文件。

## 分辨率


<b><u>目标：</u></b> 升级内部版本

<b><u>环境：</u></b> 全部

<b><u>原因：</u></b>  `con` 是 [!DNL Windows]

<b><u>分辨率:</u></b>

在 [!DNL Microsoft Windows]，则无法创建文件夹或拥有包含以下词语的扩展。

`CON, PRN, AUX, NUL, COM1, COM2, COM3, COM4, COM5, COM6, COM7, COM8, COM9, LPT1, LPT2, LPT3, LPT4, LPT5, LPT6, LPT7, LPT8, and LPT9`

这些是 [!DNL Windows]. 因此，使用这些名称命名实例时，在系统中创建具有实例名称的文件夹时升级会出错。
