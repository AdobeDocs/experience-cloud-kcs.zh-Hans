# 页面的资产令牌未被Target VEC识别

## 描述


Target VEC无法正确识别页面的资产令牌。 创建活动时，VEC会显示警告消息“在此页面上未找到资产令牌”。


## 分辨率


让我们先明白， <b>在“创作”和“交付”期间如何使用“at_property”？</b>

<b>创作 —  </b>
客户在VEC中加载其页面/SPA时，VEC会扫描页面以查找 <b>window.targetPageParams </b>函数。 如果找到此函数，则会执行该函数并尝试提取“at_property”值，并根据此值确定此属性的工作区。 如果VEC找不到此函数或此函数缺失，则VEC将默认使用默认工作区。

<b>投放 — </b>
在交付过程中，Target边缘不考虑“at_property”值的来源。 只要在请求中传递属性值，所有内容都将正常工作。


<b>在Launch中如何工作？</b>
概述了这两种操作模式，我必须解释Launch中的情况。 在Launch中，客户可以按任何所需的方式将参数添加到Launch操作，因此通常将“at_property”作为参数传递到“Fire global mbox请求”。 这非常适合交付，但在VEC中加载网站时，由于没有 <b>window.targetPageParams </b>VEC无法拾取正确的工作区。

<b>解决方法：</b>
要解决此问题并确保VEC按预期运行，一种方法是要求客户创建一个Launch自定义代码操作，以定义 <b>window.targetPageParams </b>函数。 客户必须确保在加载Target库或触发全局mbox请求之前立即执行此操作。

一种替代方法是调整VEC，以便VEC与AdobeLaunch之间存在本机集成，但这需要正确的范围界定，且实施过程将需要更多时间。



<b>解决方法的步骤：</b>

1. 创建Launch自定义代码操作，以使用“at_property”定义window.targetPageParams函数。

*window.targetPageParams = function(){return {&quot;at_property&quot; :&quot;%atproperty%&quot;};}*

2. 另外，请确保在加载Target库或触发全局mbox请求之前立即执行此操作



上述步骤应确保VEC在创建活动时正确识别属性。
