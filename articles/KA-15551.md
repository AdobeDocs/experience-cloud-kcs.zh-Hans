# 使用Campaign Data Connectors时，Analytics中为何存在数据延迟？

## 描述


使用Adobe Analytics与Adobe Campaign之间的集成时，Analytics中的数据最多会延迟20小时。  当Campaign每15分钟将数据上传到Analytics时，客户通常会关注这一点。


## 分辨率


Campaign使用Analytics数据源API将数据上传到Analytics。 该集成会在目标报表包中自动创建数据源帐户。 Campaign通过API每15分钟将数据上传到Analytics服务器。

Campaign仅使用日期信息上载数据。 数据没有时间戳。 Analytics数据源在没有时间戳的情况下，会在每天中午12点处理数据。

假设数据是今天下午4点上传的。 由于数据中没有时间戳，因此数据源直到次日中午12点才会处理文件。 第二天中午12点，数据会被处理。 添加45分钟的常规延迟，并且数据应在中午12:24后可用。 所以，大约花了23个小时来显示数据，这是预期的。

向Campaign工程团队提出了一项功能请求，以在上传到Analytics的数据中添加时间戳。 这应该可以解决问题。 目前，此程序没有ETA。
