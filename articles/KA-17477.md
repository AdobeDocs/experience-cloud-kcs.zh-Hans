<b>命名慣例</b>

使用include或include選擇性陳述式鎖定檔案時，Apache Webserver實際上並不在乎該檔案的副檔名是什麼。 以能消除衝突和混淆的名稱適當地命名它們會很有幫助。 使用的名稱會說明套用檔案的範圍，讓工作更輕鬆。 如果所有專案都命名為.conf，這會讓人感到非常困惑。 我們想要避免檔案和副檔名命名不佳。

<b>已避免的問題</b>

遵循命名慣例時，您可以避免某些相當容易犯的錯誤，這些錯誤可能會造成災難性的結果。 我們將介紹一些範例。

<u>問題範例</u>

作為網站範例（如ExampleCo），Dispatcher設定的開發人員建立了兩個設定檔案。

`/etc/httpd/conf.d/exampleco.conf`


```
`<` VirtualHost *:80`>` 

    ServerName  "exampleco"

    ServerAlias "www.exampleco.com"

    .......... SNIP ...............

    `<` IfModule mod_rewrite.c`>` 

        ReWriteEngine   on

        LogLevel warn rewrite:trace1

        Include /etc/httpd/conf.d/rewrites/exampleco.conf

    `<` /IfModule`>` 

`<` /VirtualHost`>`
```


`/etc/httpd/conf.d/rewrites/exampleco.conf`


```
RewriteRule /$ /content/exampleco/en.html `[` PT,L`]` 

RewriteRule /robots.txt$ /content/dam/exampleco/robots.txt `[` PT,L`]`
```


<b>潛在危險</b>

<b>A.檔案名稱相同</b>

若不小心將vhost檔案放入rewrites資料夾，且將rewrites檔案放入vhosts資料夾。 檔案名稱似乎已正確部署，但Apache將擲回錯誤，且問題不會立即顯現。

<b>這通常如何變成問題？</b>

如果將兩個檔案下載到相同的位置，它們可能會自行覆寫，或使部署過程變得無法區分，從而造成噩夢。

<b>B.副檔名相同，且容易自動加入</b>

檔案副檔名相同，且使用自動包含的副檔名，Apache會自動將其任何.conf檔案包含在許多預設資料夾中。

<b>這通常如何變成問題？</b>

如果將副檔名為.conf的vhost檔案放在/etc/httpd/conf.d/資料夾中，它會嘗試將它載入Apache上的記憶體（通常沒問題），但如果副檔名為.conf的重寫規則檔案放在 `/etc/httpd/conf.d/` 資料夾，則會自動納入並套用至全域，造成混淆和不想要的結果。


## 解决方法 {#resolution}


根據檔案的用途將其命名，並安全地從自動包含規則名稱空間中移除。

- 如果是虛擬主機檔案名稱，則使用 `.vhost` 作為擴充功能。
- 如果是重寫規則檔案，請將其命名為「&lt;」` site`>` _rewrite.rules` 做為尾碼和擴充功能。 此命名慣例將清楚說明該網站用於哪個網站，且這是一組重寫規則。
- 如果是IP白名單規則檔案，請將其命名為「&lt;」` description`>` _whitelist.rules` 做為尾碼和擴充功能。 此命名慣例會提供其用途的說明，並指出這是一組IP比對規則。


如果檔案被移動到不屬於的自動包含目錄中，則使用這些命名慣例可避免問題。

例如，將名為.rules、.any或.vhost的檔案放在的auto-include資料夾中 `/etc/httpd/conf.d/` 不會有任何影響。

如果部署變更請求顯示 `please deploy exampleco_rewrite.rules to production dispatchers` 部署變更的人員已經知道他們不會新增網站；他們只是更新檔案名稱所指示的重寫規則。

```
    }

    /filter {

        $include "/etc/httpd/conf.dispatcher.d/filters/ams_publish_filters.any"

        $include "/etc/httpd/conf.dispatcher.d/filters/weretail_search_filters.any"

    }

    ....SNIP....

    /cache {

        ....SNIP....
```

由於您可以看到Weretail陣列的每個區段而不具有所需的所有語法，因此請改用include陳述式。

讓我們來看看其中幾個包含的語法，以瞭解每個子包含的外觀如何 `/etc/httpd/conf.dispatcher.d/vhosts/weretail_publish_vhosts.any`：



如您所見，這是一個新的以行分隔的網域名稱清單，應從此陣列而不是其他陣列呈現。

接下來，讓我們看看 `/etc/httpd/conf.dispatcher.d/filters/weretail_search_filters.any`：
