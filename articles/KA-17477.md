# 配置文件说明 | AEM

## 描述 {#description}

<b>环境</b>
Experience Manager
<b>问题/症状</b>
本文档将划分并解释在Adobe Managed Services中设置的标准构建调度程序服务器中部署的每个配置文件。 它们的使用、命名规范等。

<b>命名约定</b>

使用include或includeoptional语句定位文件时，Apache Webserver实际上并不关心文件的文件扩展名是什么。 使用可消除冲突和混淆的名称正确命名这些值，可帮您大忙。 使用的名称将描述文件的应用范围，使其更轻松。 如果所有内容都名为.conf，这会让人很困惑。 我们希望避免文件和扩展名名称不佳。

以下是典型AMS配置的调度程序中使用的不同自定义文件扩展名和命名约定的列表。

<b>避免的问题</b>

遵循命名规范时，您可以避免一些非常容易的错误，这些错误可能会产生灾难性的结果。 我们将介绍几个例子。

<u>问题示例</u>

例如，ExampleCo的两个配置文件是由调度程序配置的开发人员创建的。

`/etc/httpd/conf.d/exampleco.conf`


```
`<` VirtualHost *:80`>` 

    ServerName  "exampleco"

    ServerAlias "www.exampleco.com"

    .......... SNIP ...............

    `<` IfModule mod_rewrite.c`>` 

        ReWriteEngine   on

        LogLevel warn rewrite:trace1

        Include /etc/httpd/conf.d/rewrites/exampleco.conf

    `<` /IfModule`>` 

`<` /VirtualHost`>`
```


`/etc/httpd/conf.d/rewrites/exampleco.conf``>`


```
RewriteRule /$ /content/exampleco/en.html `[` PT,L`]` 

RewriteRule /robots.txt$ /content/dam/exampleco/robots.txt `[` PT,L`]`
```


<b>潜在危险</b>

<b>A.文件名相同</b>

如果vhost文件意外放入rewrites文件夹，而rewrites文件被放入vhosts文件夹中。 它似乎按文件名正确部署，但Apache将引发错误，问题不会立即显示。

<b>这通常是如何成为问题的？</b>

如果将两个文件下载到同一位置，则它们可能会自行覆盖或使其无法区分，从而使部署过程成为噩梦。

<b>B.文件扩展名相同，并且容易自动包含</b>

文件扩展名是相同的，使用的是自动包含的扩展名，Apache将在其许多默认文件夹中自动包含任何.conf文件。

<b>这通常是如何成为问题的？</b>

如果扩展名为.conf的vhost文件放在/etc/httpd/conf.d/文件夹中，它将尝试将其加载到Apache上的内存中，这通常是正常的，但如果扩展名为.conf的重写规则文件放在 `/etc/httpd/conf.d/` 文件夹，则会自动包含该变量并在全局范围内应用，从而导致结果混乱和不希望。


## 解决方法 {#resolution}


根据文件的操作命名文件，并安全地从自动包含规则命名空间中命名。

- 如果是虚拟主机文件名，则使用 `.vhost` 作为扩展。
- 如果是重写规则文件，请将其命名为“&lt;”` site`>` _rewrite.rules` 作为后缀和扩展名。 此命名约定将明确其用于哪个站点，并且是一组重写规则。
- 如果是IP白名单规则文件，请将其命名为“&lt;` description`>` _whitelist.rules` 作为后缀和扩展名。 此命名约定将对其用途进行一些描述，并且是一组IP匹配规则。


如果文件被移动到它不属于的自动包含目录中，使用这些命名约定将避免出现问题。

例如，将名为.rules、.any或.vhost的文件放入的auto-include文件夹中 `/etc/httpd/conf.d/` 不会有任何影响。

如果部署更改请求显示 `please deploy exampleco_rewrite.rules to production dispatchers` 部署更改的人员已经知道他们没有添加新站点；他们只是在更新文件名中指示的重写规则。

C. AMS虚拟主机包括

/Directory

....SNIP....

Include /etc/httpd/conf .d /rewrites/weretail_rewrite .rules

/IfModule /VirtualHost
```

D. AMS场包括

的 ``<` FILENAME`>` _farm.any` files will have include statements for each file based on where they need to be included in the farm file. Here is an example syntax of a ``&lt;` FILENAME`>` _farm.any` 文件作为良好引用：

    }

    /filter {

        $include "/etc/httpd/conf.dispatcher.d/filters/ams_publish_filters.any"

        $include "/etc/httpd/conf.dispatcher.d/filters/weretail_search_filters.any"

    }

    ....SNIP....

    /cache {

        ....SNIP....

        /rules {

            $include "/etc/httpd/conf.dispatcher.d/cache/ams_publish_cache.any"

        }

        ....SNIP....

        /allowedClients {

                /glob "*.*.*.*"

如您所见，weretail场的每个部分都使用include语句，而不是包含所需的所有语法。

下面我们看一下其中一些包含的语法，以了解每个子包含的外观 `/etc/httpd/conf.dispatcher.d/vhosts/weretail_publish_vhosts.any`:

如您所见，这是一个以行分隔的域名新列表，应从此场呈现给其他场。

/401 { /type "allow" /method "POST" /path "/bin/weretail/search/" /extension "html" }
```
