# 如何优化Oak Lucene索引以缩短索引时间

## 描述


优化AEM Oak lucene索引以缩短重新索引时间。
<br><br>环境<br><br>
AEM 6.0 + Oak 1.0.16或更高版本AEM 6.1和AEM 6.2


## 分辨率


您可以优化Oak Lucene索引，方法是通过使用“includedPaths”、“excludedPaths”、“name”和“indexPath”等属性来优化oak: QueryIndexDefinition。

- “includedPaths”和“excludedPaths”属性限制索引在重新索引时遍历的路径以及它侦听的索引更新路径。
- “name”和“indexPath”属性使索引日志更容易理解，因为它会使用索引的路径或名称标记日志消息。  例如：



| `30.01.2017 14:07:12.934 *DEBUG* pool-8-thread-2 org.apache.jackrabbit.oak.plugins.index.lucene.LuceneIndexEditor damassetlucene(` `/oak` `:index` `/damAssetLucene` `) = Indexed 9000 nodes...` |
| --- |


<b>优化索引的步骤</b>:

1. 转到http://*aem-host:port*/crx/de/index.jsp ，以管理员身份登录
2. 浏览到/oak:index
3. 将下面突出显示的属性添加到相应的索引中。  的 <b>includedPaths</b> 和 <b>excludedPaths</b> 属性应为多值字符串（字符串）
4. 单击“[!UICONTROL 全部保存]“ ”
