# 如何优化Oak Lucene索引以减少索引时间


使用includedPaths和excludedPaths属性优化Oak Lucene索引。

## 描述 {#description}


### <b>环境</b>

- Adobe Experience Manager 6.0 + Oak 1.0.16或更高版本
- Adobe Experience Manager 6.1和6.2




### <b>问题/症状</b>

优化AEM Oak Lucene索引以减少重新索引时间。


## 解决方法 {#resolution}


您可以通过微调Oak来优化Oak Lucene索引：` QueryIndexDefinition` 具有以下属性 `includedPaths`， `excludedPaths`， `name` 和 `indexPath`.

- `includedPaths` 和 `excludedPaths` 属性会限制索引在重新索引时遍历的路径，以及它侦听索引更新的路径。
- `name` 和 `indexPath` 属性使索引日志更容易理解，因为它们会使用索引的路径或名称标记日志消息。 例如：



| `30.01.2017 14:07:12.934 *DEBUG* [ pool-8-thread-2]  org.apache.jackrabbit.oak.plugins.index.lucene.LuceneIndexEditor [ damassetlucene(` `/oak` `:index` `/damAssetLucene` `)]  => Indexed 9000 nodes...` |
| --- |


<b>优化索引的步骤</b>：

1. 转到 *http://aem-host:port/crx/de/index.jsp*&#x200B;并以管理员身份登录。
2. 浏览至 `/oak:index`.
3. 添加下面高亮显示的属性 `[` 1`]`  到相应的索引。 此` includedPaths `和` excludedPaths `属性应为 `multi-value String (String[ ] )`.
4. 单击 <b>全部保存</b> 在CRXDe中。
