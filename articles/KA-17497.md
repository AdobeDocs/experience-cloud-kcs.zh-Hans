# 了解AEM Dispatcher缓存哪些请求

## 描述 {#description}

<br>问题<br><br>
需要确定AEM Dispatcher为何没有缓存HTTP请求/响应
<br><br>环境<br><br>
AEM Dispatcher 4.3.3


## 解决方法 {#resolution}


本文档介绍了影响通过AEM Dispatcher模块的请求和响应的缓存能力的最重要方案。 未涵盖所有缓存规则。

<b>HTTP请求和缓存规则</b>

要使调度程序可缓存HTTP请求，它必须遵循以下规则：

1. URL是绝对的(不适用于。 或……)，具有文件扩展名，且URL中的文件名可用作文件系统上的文件。
   <b>相关日志消息：</b>


   ```
   URI not canonical: %s.    
   Unable to map URI to file: %s.    
   cache file path too long    
   temporary file path too long    
   request URL has no extension
   ```



   <b>示例：</b>


   | `/content/test` | 未缓存 |
   | --- | --- |
   | `/content/test.html` | cached |


2. 第一个文件扩展名后没有斜杠。<br>

   <b>相关日志消息：</b>


   ```
   request URL has trailing slash
   ```



   <b>示例：</b>


   | `/content/test.html/` | 未缓存 |
   | --- | --- |
   | `/content/test.html/foo.jpg` | 未缓存 |
   | `/content/test.html` | cached |


3. 它使用HTTPGET或HEAD方法。<br>

   <b>相关日志消息：</b>


   ```
   request method is neither GET nor HEAD
   ```



   <b>示例：</b>


   | `HEAD /content/test.html HTTP/1.1` | cached |
   | --- | --- |
   | `GET /content/dam/test.jpg HTTP/1.1` | 未缓存 |
   | `POST /content/test.html HTTP/1.1` | 未缓存 |


4. 调度程序中定义的/CACHE = /RULES拒绝该请求。 任何场配置。
   <b>相关日志消息：</b>


   ```
   URI not  in cache rules: %s request URL not in cache rules
   ```



   <b>示例：</b>


   | <br><br>    ```<br>    Farm has /cache = /rules and a request is received for /content/test.html<br>        /cache<br>        {<br>         ...<br>         /rules<br>         {<br>          /0001 {<br>          type "deny"<br>          glob "*"<br>          }<br>         }<br>        }<br>    ```<br><br> | 未缓存 |
   | --- | --- |
   | <br><br>    ```<br>    Farm has /cache = /rules and a request is received for /content/test.html<br>        /cache<br>        {<br>         ...<br>         /rules<br>         {<br>          /0001 {<br>          type "allow"<br>          glob "*"<br>          }<br>         }<br>        }<br>    ```<br><br> | cached |


5. HTTP请求不包含授权标头，或者包含标头，但在场配置中，/ALLOWAUTHORIZED设置为1。
   <b>相关日志消息：</b>


   ```
   request contains authorization
   ```



   <b>示例：</b>


   | HTTP请求：<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Authorization: Basic YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | 未缓存 |
   | --- | --- |
   | HTTP请求：<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Authorization: Basic YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “1”    }<br>    ```<br><br> | cached |


6. HTTP请求在COOKIE标头中包含LOGIN-TOKEN或AUTHORIZATION COOKIE，或者它包含（或同时包含）这些COOKIE中的任意一个（或两个），并且/ALLOWAUTHORIZED在场配置中设置为1。
   <b>相关日志消息：</b>


   ```
   request contains authorization
   ```



   <b>示例：</b>


   | HTTP请求：<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: login-token=…<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | 未缓存 |
   | --- | --- |
   | HTTP请求：<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: authorization=YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | 未缓存 |
   | HTTP请求：<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: login-token=…<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “1”    }<br>    ```<br><br> | cached |


7. URL中有一个QUERYSTRING参数，允许通过FARM的/IGNOREURLPARAMS配置忽略该参数。
   <b>相关日志消息：</b>


   ```
   request contains a query string
   ```



   <b>示例：</b>


   | HTTP请求：<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    Farm /cache configuration:<br>        /ignoreUrlParams {     /0001 { /type “allow” /glob “*” }    }<br>    ```<br><br> | cached |
   | --- | --- |
   | HTTP请求：<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    Farm /cache configuration:<br>        /ignoreUrlParams {     /0001 { /type “deny” /glob “*” } <br>           /0001 { /type “allow” /glob “test” }    }<br>    ```<br><br> | cached |
   | HTTP请求：<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /ignoreUrlParams {<br>         /0001 { /type “deny” /glob “*” } <br>           /0001 { /type “allow” /glob “q” }    }<br>    ```<br><br> | 未缓存 |



<b>HTTP响应和缓存规则：</b>

如果满足以下条件，则可缓存从AEM返回的HTTP响应：

1. 调度程序能够从定义的“RENDERS”之一发送和接收200 OK响应。

   <b>注意：</b> 如果否 [/timeout](https://experienceleague.adobe.com/docs/experience-manager-dispatcher/using/configuring/dispatcher-configuration.html?lang=zh-Hans) 在/renders中设置，或者将其设置为0，则它将永远等待与AEM实例的连接，即使实例关闭也是如此。
   <b>相关日志消息：</b>


   ```
   Unable to send request to remote server. Unable to receive response from remote server. Remote server returned: %s No backend available.
   ```



   <b>示例：</b>


   | HTTP请求：<br><br>    200 OK | cached |
   | --- | --- |
   | HTTP请求：<br><br>    404页面未找到 | 未缓存 |


2. 响应中不存在以下任何响应标头：

   - `Dispatcher: no-cache`
   - `Cache-control: no-cache`
   - `Pragma: no-cache`

   <b>相关日志消息：</b>


   ```
   Backend forbids caching: %s, sent: %s" response contains no_cache
   ```



   <b>示例：</b>


   | HTTP响应：<br><br>    200 OK<br><br>    调度程序：无缓存 | 未缓存 |
   | --- | --- |
   | HTTP响应：<br>    <br><br>    200 OK<br><br>    缓存控制：无缓存 | 未缓存 |
   | HTTP响应：<br><br>    200 OK<br><br>    Pragma:无缓存 | 未缓存 |
   | HTTP响应：<br><br>    200 OK<br> | cached |


3. 内容长度值大于零字节。
   <b>相关日志消息：</b>


   ```
   response content length is zero
   ```



   <b>示例：</b>


   | HTTP响应：<br><br>    200 OK<br><br>    Content-Length:0 | 未缓存 |
   | --- | --- |
   | HTTP响应：<br><br>    200 OK<br><br>    Content-Length:3120 | cached |
